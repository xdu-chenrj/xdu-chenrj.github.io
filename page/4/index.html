磁盘IO分为两步，一步是准备数据，一步是获取数据

unix IO模型

阻塞IO （同步阻塞）【两阶段都是阻塞的】

非阻塞IO （同步非阻塞）【准备数据阶段不阻塞，获取数据阶段阻塞】

多路复用IO（同步非阻塞）【selector阻塞，别的事件不阻塞，获取事件阻塞】

信号驱动IO

异步IO（异步非阻塞）

阻塞IO和多路复用IO的区别：

阻塞IO只能通过特定的时间结束阻塞，而多路复用IO可以通过任意注册的事件来结束阻塞

阻塞IO是具体的阻塞，多路复用是selector阻塞，selector可以检测多种事件，返回一批事件，然后在一个循环中处理这些事件。

阻塞和非阻塞的区别

阻塞：调用线程只有在得到结果之后才会返回。（线程发起read需要等待数据）

非阻塞：在不能立刻得到结果之前，不会阻塞当前线程。（线程发起read不需要等待直接返回）

同步和异步的区别

同步：线程自己获取结果

异步：线程自己不去获取结果，而是由其他线程送结果（至少两个线程）

Java IO模型

BIO （同步阻塞IO）

NIO（同步非阻塞IO）
